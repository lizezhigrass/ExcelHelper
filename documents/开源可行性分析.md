# Excel 快捷查询助手 — 通用性分析与开源价值评估

## 一、项目概览

**Excel 快捷查询助手**是一个 Windows 桌面工具，以系统托盘方式常驻后台。用户在 Excel 中选中某行，通过全局快捷键触发查询，结果以弹窗表格呈现，可双击一键回写目标列。

| 项目 | 详情 |
|------|------|
| 语言 | Python 3.x |
| UI 框架 | PyQt5 |
| 配置格式 | YAML（`config.yaml`） |
| 打包方式 | PyInstaller（`build.bat` + `.spec`） |
| 代码规模 | ~250KB，约 3500 行有效代码 |

---

## 二、架构分析（通用性评估核心）

### 2.1 三层解耦架构 ✅

项目采用了非常清晰的三层结构（`core/mapping.py` 中有完整文档注释）：

```
快捷键触发层（table_modules）
    ↓  指向
映射组层（mapping_groups）—— 数据如何流动（读哪列、写哪列、显示什么）
    ↓  引用
交互模块层（interaction_modules）—— 窗口长什么样
    ↓  引用
查询后端层（query_backends/）—— 数据从哪来
```

这种解耦使得**同一套 UI 框架可以服务于完全不同的业务数据**，是通用性的关键基础。

### 2.2 插件化查询后端 ✅

`QueryBase` 抽象基类 + `@register_backend("type_name")` 装饰器 构成了一套干净的插件机制：

| 已内置后端类型 | 说明 |
|--------------|------|
| `vector_api` | REST API 方式调向量检索服务 |
| `qdrant_vector` | 直连 Qdrant 向量数据库（gRPC/HTTP） |
| `pg_plan` | 直连 PostgreSQL（计划历史专用） |
| `pg_spare_parts` | 直连 PostgreSQL（备件专用） |
| `pg_material_api` | 通过 REST API 查 PG 物料库 |
| `excel_file` | 内存加载本地 Excel 文件，模糊/正则检索 |
| `sql_table` | 通用 SQL 后端（PostgreSQL + SQLite，ILIKE/精确匹配） |

> **关键设计**：新增任意数据源只需继承 `QueryBase`，实现 4 个方法，加一个装饰器，无需改动任何上层代码。

### 2.3 四种交互模式 ✅

| base_type | 功能描述 |
|-----------|---------|
| `popup_table` | 弹窗展示结果 + UI 控件（复选框/下拉）动态过滤 |
| `search_panel` | 多字段搜索面板，Excel 数据预填 |
| `auto_fill` | 批量自动填充（支持多行框选，进度条展示） |
| `display_panel` | 完全自定义的组件面板（文本框/日期/数字/下拉/复选） |

### 2.4 全图形化设置界面 ✅

`ui/settings_window.py`（160KB，最大单文件）实现了完整的 GUI 配置管理，用户无需手写 YAML 即可：
- 新建/编辑/删除 查询模块、映射组、交互模块
- 拖动排序、即时应用修改、连接状态反馈

## 三、通用性判断

### ✅ 高度通用的部分

1. **查询后端插件系统**：`sql_table` 后端本身就是通用的，任意 PostgreSQL/SQLite 表都可接入，无业务耦合。
2. **Excel 文件后端**：支持多文件源、目录遍历、自动列映射，通用性强。
3. **交互模式设计**：四种交互模式覆盖了大多数 Excel 辅助查询场景。
4. **配置驱动**：整个行为由 YAML 文件驱动，零代码适配不同业务。
5. **自动填充批处理**：支持 Excel 框选多行批量回写，是较罕见的实用功能。

### ⚠️ 当前仍存在的业务耦合

1. **`config.yaml` 中暴露了生产环境的私有数据**：
   - 内网 IP 地址（`10.116.231.70`）
   - 数据库明文密码（`Passwd_0f_Plan`）
   - Qdrant API Key
   - 具体业务表名（`material_plan`、`送货明细` 等）
   - 内网文件路径（`G:/物资备件相关/...`）

2. **部分后端是业务专用的**：
   - `pg_plan_query.py`、`pg_spare_parts_query.py`、`pg_material_query.py` 三个后端针对特定数据库 schema 硬编码了 SQL 查询逻辑。
   - 通用的 `sql_table` 后端已可替代它们，说明这些是历史遗留的定制化实现。

3. **代码注释和 UI 文字全为中文**：对非中文社区的使用者有一定门槛。

## 四、与同类工具对比

| 工具 | 定位 | 与本项目差异 |
|------|------|------------|
| Excel VLOOKUP/Power Query | 内置公式/查询 | 仅限 Excel 内部数据；无法连接外部 DB/向量库 |
| Microsoft Power Automate | 低代码自动化 | 侧重流程自动化，非实时查询辅助 |
| VBA 宏 | Excel 内嵌脚本 | 无法连接向量数据库；维护成本高 |
| 本项目 | 外部快捷查询助手 | **跨数据源**（SQL/向量/文件）+ **系统托盘常驻** + **无需改动 Excel 文件** |

**核心差异化价值**：在不修改 Excel 文件本身的情况下，通过快捷键从任意外部数据源（包括向量语义检索）查询数据并回写，这在开源工具中几乎没有直接竞品。

## 五、开源价值评估

### 价值亮点 ⭐⭐⭐⭐

- 解决了**「Excel 作为工作表 + 外部数据库作为数据源」**这一极其普遍的企业场景
- 插件化架构**对二次开发友好**，新增数据源成本极低
- 向量语义检索集成（Qdrant + BGE 嵌入）在同类工具中较为前沿
- 支持打包为 `.exe` 分发，非技术用户也可使用
- 代码质量较高，有完整的 docstring、类型注解、日志系统

### 主要障碍（开源前需处理）

| 问题 | 严重程度 | 解决方案 |
|------|---------|---------|
| `config.yaml` 含私有凭据 | 🔴 必须处理 | 提供 `config.example.yaml` 替代，原文件加入 `.gitignore` |
| 业务专用后端混杂 | 🟡 建议处理 | 将 `pg_plan_query.py` 等标注为示例，推荐用 `sql_table` 替代 |
| 无 README / 文档 | 🟡 建议处理 | 编写安装、配置、后端扩展指南 |
| 无示例配置 | 🟡 建议处理 | 提供 SQLite demo 数据 + 示例配置，让用户 10 分钟跑通 |
| 全中文代码注释 | 🟢 视目标受众 | 若面向国际社区，需翻译；面向中文社区则不必 |

## 六、总结与建议

### 总体判断

**本项目通用性强，具有开源价值。**

其核心框架（三层架构 + 插件后端 + YAML 配置驱动）是完全与业务无关的，完全可以作为一个通用的「Excel 快捷数据查询与回写工具」供他人使用。

### 建议的开源准备步骤

1. **立即执行（安全问题）**：
   - 将 `config.yaml` 加入 `.gitignore`
   - 提供 `config.example.yaml`（使用占位值替代真实凭据）

2. **建议执行（用户体验）**：
   - 编写 `README.md`：项目简介、快速上手、配置参考、后端扩展指南
   - 提供一个内置 SQLite 数据库 + 示例配置，使新用户无需配置即可看到效果
   - 将业务专用后端（`pg_plan`、`pg_spare_parts`）移至 `examples/` 目录并注明"仅作示例"

3. **可选（扩大受众）**：
   - 英文化核心注释（或提供双语文档）
   - 增加更多内置后端（MySQL、MongoDB、CSV 文件等）
   - 添加单元测试

GitHub，建议分类标签：`excel`、`productivity`、`python`、`pyqt5`、`qdrant`、`vector-search`、`windows`
